
  Splunk is one of the leading SIEM solutions in the market that provides the ability to collect,
  analyze and correlate network and machine logs in real-time.

  Splunk has three main components:
    Forwarder:
      Installed on the endpoint to be monitored, forwarder collects data and sends it back
      to the Splunk instance. Minimally impacts endpoint performance.
    Indexer:
      Processes data received from Forwarders into key:value pairs and determines data types.
    Search Head:
      Where users can search the indexed data using terms or Splunk Search Processing Language.

SPLUNK 201

  Start by clicking on "Search and Reporting" on the left panel.

  Interesting log sources:
    wineventlog: It contains Windows Event logs
    winRegistry: It contains the logs related to registry 
                 creation / modification / deletion etc.
    XmlWinEventLog: It contains the sysmon event logs. It is a very important 
                    log source from an investigation point of view.
    fortigate_utm: It contains Fortinet Firewall logs
    iis: It contains IIS web server logs
    Nessus:scan: It contains the results from the Nessus vulnerability scanner.
    Suricata: It contains the details of the alerts from the Suricata IDS. 
              This log source shows which alert was triggered and what caused the alert 
              to get triggered— a very important log source for the Investigation.
    stream:http: It contains the network flow related to http traffic.
    stream: DNS: It contains the network flow related to DNS traffic.
    stream:icmp: It contains the network flow related to icmp traffic.

  All available log and other sources can be viewed in the 
  "Data Summary" tab found on the search tab.

  Before searching, make sure the timeframe is correct, and 
  the search mode is set to the intended value.

  Checking for reconnaisance activity:
    Look for IP addresses with heavy activity. When these are located, check things 
    like user agents, POST requests, URI's for suspicious activity. 

  Checking for exploitation activity:
    Search for requests by IP count using the given search query:
      index=botsv1 imreallynotbatman.com sourcetype=stream* | stats count(src_ip) as Requests by src_ip | sort - Requests
    
    Useful information can be found by searching for traffic for a specific destination IP, then
    checking from what source IP addresses traffic is coming from.
    
    Analyzing HTTP traffic:
      Try checking and filtering by HTTP method.

      Check interesting fields like "src_ip", "form_data", "http_user_agent", and "uri"

      Use the following query for a convenient table with useful info (make sure to filter for
      POST requests):
          |  table _time uri src_ip dest_ip form_data

      Extract password credentials with the following query:
        index=botsv1 sourcetype=stream:http dest_ip="192.168.250.70" http_method=POST form_data=*username*passwd* 
        | rex field=form_data "passwd=(?<creds>\w+)"  
        | table src_ip creds




______
TABLES

  To create a table, use the keyword "table" and follow it with the intended columns

  Examples:
    table _time uri src_ip dest_ip form_data

_____
REGEX

  To use regex, use the keyword "rex", type the field=[FIELD] to search in, then add a regex 
  in quotes. Extractions can be performed using (?<EXTRACTION>) which can then be accessed
  subsequently in the query.

  Examples:
    rex field=Local ".*:(?<port>.*)"
    rex field=form_data "passwd=(?<creds>\w+)" | table src_ip creds
