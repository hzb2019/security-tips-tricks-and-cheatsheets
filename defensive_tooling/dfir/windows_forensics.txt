GENERAL REGISTRY INFO

  "The Windows Registry is a collection of databases that contains the 
  system's configuration data."

  Registry entries are separated into "hives":
  https://docs.microsoft.com/en-us/windows/win32/sysinfo/registry-hives#:~:text=Registry%20Hives.%20A%20hive%20is%20a%20logical%20group,with%20a%20separate%20file%20for%20the%20user%20profile.

  There are five root keys on every Windows System:
    HKEY_CURRENT_USER (HKCU, contains user folders, screen colors, control panel settings)
    HKEY_USERS (HCU, HKCU is a subkey of this, contains all actively loaded user info)
    HKEY_LOCAL_MACHINE (HKLM, contains config info pertaining to the computer)
    HKEY_CLASSES_ROOT (HKCR, helps open the correct files through windows explorer)
      This is a subkey of HKLM\Software.
      Starting in Windows 2000, it started being stored under both HKLM and HKCU keys.
      HKLM keys apply to everybody on the system, HKCU keys override those for user-specific
      preferences. 
      HKCR merges the information from HKLM and HKCU, and even merges them for softwares in
      earlier versions of Windows. To change settings for the interactive user, changes must
      be made to HKCU\Software\Classes instead of under HKCR. To change default settings, 
      change HKLM\Software\Classes (writing to HKCR will write here too).

      If you write a value to HKCR and the key already exists in HKCU\Software\Classes, it will
      try to then write it to HKLM\Software\Classes.
    HKEY_CURRENT_CONFIG ()
    https://docs.microsoft.com/en-US/troubleshoot/windows-server/performance/windows-registry-advanced-users

  Key details and locations:
    Live registry keys can be viewed when running regedit.exe, or can be read from a disk image 
    at C:\Windows\System32\Config. This will contain most hives and include:
      DEFAULT (HCU\DEFAULT)
      SAM (HKLM\SAM)
      SECURITY (HKLM\Security)
      SOFTWARE (HKLM\Software)
      SYSTEM (HKLM\System)
    If user-specific information is needed:
      NTUSER.DAT (mounts to HKCU when logged in, found at C:\Users\[USERNAME]\)
      USRCLASS.DAT (HKCU\Software\CLASSES when logged in, found at C:\Users\[USERNAME]\AppData\Local\Microsoft\Windows)
    Amcache hive for recently run programs:
      C:\Windows\AppCompat\Programs\Amcache.hve

  Logs and Backups:
    Transaction logs (saved as .LOG, .LOG1, .LOG2 ...) contains changelogs for registry keys and 
    are saved in the same directory as the hives they track. Sometimes can contain changes that 
    still need to be implemented to the keys.

    Registry backups contain copies of the registry keys saved in System32\Config and are
    saved every 10 days to System32\Config\RegBack.

  Tools for collecting Registry information:
    KAPE: https://www.kroll.com/en/services/cyber-risk/incident-response-litigation-support/kroll-artifact-parser-extractor-kape
    Autopsy: https://www.autopsy.com/
    FTK Imager: https://www.exterro.com/ftk-imager 
      "Another way you can extract Registry files from FTK Imager is through the Obtain 
      Protected Files option. This option is only available for live systems and is highlighted 
      in the screenshot below. This option allows you to extract all the registry hives to a 
      location of your choosing. However, it will not copy the Amcache.hve file, which is often 
      necessary to investigate evidence of programs that were last executed.

  Tools for browsing collected Registry information:
    Registry Viewer: https://accessdata.com/product-download/registry-viewer-2-0-0
      Only loads one hive at a time, cannot take into account transaction logs.
    Zimmerman's Registry Explorer: https://ericzimmerman.github.io/#!index.md
    RegRipper: https://github.com/keydet89/RegRipper3.0
      Takes in a hive and processes forensically interesting keys into a text file.
      Also does not take into account transaction logs.

INVESTIGATING THE REGISTRY

  System Information:
    OS Version: SOFTWARE\Microsoft\Windows NT\CurrentVersion
    Control Sets (Hives containing startup configurations):
      SYSTEM\ControlSet001 (Configuration loaded at boot)
      SYSTEM\ControlSet002 (Last known good configuration)

      SYSTEM\Select\Current (Control set being used as CurrentControlSet)
      SYSTEM\Select\LastKnownGood (Another way to find last known good config)

      NTUSER.DAT\Software\Microsoft\Windows\Currentversion\Explorer\UserAssist\[USER GUID]\Count

      SYSTEM\CurrentControlSet\Control\Session Manager\AppCompatCache

      Amcache.hve\Root\File\{Volume GUID}\
    Background/Desktop Activity Monitor (BAM/DAM, last run programs, full path, exec time):
      SYSTEM\CurrentControlSet\Services\bam\UserSettings\{SID}
      SYSTEM\CurrentControlSet\Services\dam\UserSettings\{SID}

  External Devices:
    USB Device Identification:
      SYSTEM\CurrentControlSet\Enum\USBSTOR
      SYSTEM\CurrentControlSet\Enum\USB

      SYSTEM\CurrentControlSet\Enum\USBSTOR\Ven_Prod_Version\USBSerial#\Properties\[DEVICE ID]\[TIME]

        0064	First Connection time

    USB Device Volume Name:
      SOFTWARE\Microsoft\Windows Portable Devices\Devices

FILE SYSTEMS

  File Allocation Table (FAT):
    Creates a table that indexes the location of bits that are allocated to different files.
    Uses three data structures:
      Clusters: Basic storage unit. Each file stored is a group of clusters containing the file data.
      Directory: Contains info including file names, starting clusters, and filename length.
      FAT: A linked list of all clusters.

    FAT12, FAT16, FAT32 all were developed to provide more storage cluster space over
    the original FAT(8), allowing for address spaces of 2^12, 2^16, and 2^28 clusters respectively.
    FAT16 and FAT32 are limited by a maximum file size of 4GB - 1. 

    exFAT was developed by Microsoft to support FAT for storage devices that didn't need the
    extra security systems in the NTFS storage format. Supports clusters between 4KB to 32MB,
    and can store up to 128PB.

  NTFS:
    Introduced in 1993 with Windows NT 3.1. Includes security features and improved reliability 
    and recovery features.

    Journaling:
      Contains a $LOGFILE in the volume root directory that logs changes to metadata and can 
      help the system to recover from a crash or data movement due to defragmentation.

    Access Controls: User-specific permissions for files/directories.

    Volume Shadow Copy: Keeps track of changes to files and can revert to previous copies.

    Alternate Data Streams: Allows multiple data streams to be saved to the same file.

    Master File Table: 
      A structured database tracking objects stored in the volume.

      $MFT: First record in the volume. Volume Boot Record (VBR) points to this. Has info 
      about the clusters where all other objects are stored. Contains directory of all files.

      $LOGFILE: Stores transactional logging of filesystem. Maintains system integrity if a crash 
      occurs.

      $UsnJrnl: Update Sequence Number (USN) journal, aka Change Journal. Keeps info about all
      files changed in the system and the reason for the change. Present in $Extend record.

      Use Eric Zimmerman's MFT Explorer to browse files. Basic usage for parsing $MFT file:
        MFTECmd.exe -f [PATH-TO-$MFT-FILE] --csv [PATH-TO-SAVE-RESULTS-IN-CSV]

EVIDENCE OF EXECUTION

  Windows Prefetch Files: C:\Windows\Prefetch
    Basically a cache for program data so they can be run faster. Has a .pf extension.
    Includes the following columns of information:
      Note, SourceFilename, SourceCreated, SourceModified, SourceAccessed, ExecutableName,
      Hash,Size, Version, RunCount, LastRun, PreviousRun0, PreviousRun1, PreviousRun2, 
      PreviousRun3, PreviousRun4, PreviousRun5, PreviousRun6, Volume0Name, Volume0Serial, 
      Volume0Created, Volume1Name, Volume1Serial, Volume1Created, Directories, FilesLoaded, 
      ParsingError
    Use EZ's Prefetch Parser (WxTCmd.exe) for parsing:
      PECmd.exe -f [path-to-Prefetch-files] --csv [path-to-save-csv]
      PECmd.exe -d [path-to-Prefetch-directory] --csv [path-to-save-csv]

  Windows 10 Timeline: C:\Users\[username]\AppData\Local\ConnectedDevicesPlatform\{randomfolder}\ActivitiesCache.db
    A SQLite database that stores recently used applications and files.
    Includes the following columns of information:
      Id, ActivityTypeOrg, ActivityType, Executable, DisplayText, ContentInfo, Payload,
      ClipboardPayload, StartTime, EndTime, Duration, LastModifiedTime, LastModifiedOnClient,
      OriginalLastModifiedOnClient, ExpirationTime, CreatedInCloud, IsLocalOnly, ETag,
      PackageIdHash, PlatformDeviceId, DevicePlatform, TimeZone
    Use EZ's WxTCmd.exe for parsing:
      WxTCmd.exe -f [path-to-timeline-file] --csv [path-to-save-csv]

  Windows Jump Lists: C:\Users\[username]\AppData\Roaming\Microsoft\Windows\Recent\AutomaticDestinations
    The thing that shows your recently used files when right clicking an icon in the taskbar.
    Includes the following columns of information:
      SourceFile, SourceCreated, SourceModified, SourceAccessed, AppId, AppIdDescription,
      DestListVersion, LastUsedEntryNumber, MRU, EntryNumber, CreationTime, LastModified,
      Hostname, MacAddress, Path, InteractionCount, PinStatus, FileBirthDroid, FileDroid,
      VolumeBirthDroid, VolumeDroid, TargetCreated, TargetModified, TargetAccessed, FileSize, 
      RelativePath, WorkingDirectory, FileAttributes, HeaderFlags, DriveType, VolumeSerialNumber,
      VolumeLabel, LocalPath, CommonPath, TargetIDAbsolutePath, TargetMFTEntryNumber,
      TargetMFTSequenceNumber, MachineID, MachineMACAddress, TrackerCreatedOn, ExtraBlocksPresent,
      Arguments, Notes
    Use EZ's JLECmd.exe for parsing:
      JLECmd.exe -f [path-to-Jumplist-file] --csv [path-to-save-csv]

FILE/FOLDER KNOWLEDGE

  Shortcut Files:
    These contain first and last opened times of the file and its path.
    C:\Users\<username>\AppData\Roaming\Microsoft\Windows\Recent\
    C:\Users\<username>\AppData\Roaming\Microsoft\Office\Recent\
    Use EZ's Lnk Explorer (LECmd.exe) for parsing:
      LECmd.exe -f <path-to-shortcut-files> --csv <path-to-save-csv>

  IE/Edge History: C:\Users\<username>\AppData\Local\Microsoft\Windows\WebCache\WebCacheV*.dat
    Can be explored using Autopsy, by selecting logical files as the data source type, then the 
    volume folder to examine and selecting "Recent Activity" in the Configure Ingest screen. 
    Checking "Web History" under "Data Artifacts" on the left panel will have the relevant 
    information.

  Jump Lists:
    Same as in Evidence of Execution section, info here can identify last executed programs 
    and last opened files.
