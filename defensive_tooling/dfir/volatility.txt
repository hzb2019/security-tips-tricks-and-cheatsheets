MEMORY EXTRACTION

  "Extracting a memory dump can be performed in numerous ways, varying based on the requirements 
  of your investigation. Listed below are a few of the techniques and tools that can be used to 
  extract a memory from a bare-metal machine." -THM
    - FTK Imager
    - Redline
    - DumpIt.exe
    - win32dd.exe / win64dd.exe
    - Memoryze
    - FastDump

  Extractions on bare metal hosts can be time intensive.

  Memory can be extracted relatively easily from VM's by collecting the virtual memory file
  from the host machine's drive. The following are common memory file formats.
    VMWare - .vmem
    Hyper-V - .bin
    Parallels - .mem
    VirtualBox - .sav file *this is only a partial memory file

PLUGINS

  Because Volatility3 does not use profiles to specify OS, when plugins are used they need to 
  be prefixed with the OS. [windows/linux/mac].[plugin]

_________________
IMAGE INFORMATION

  Not all memory files will come with information about the system they describe. In these
  cases, volatility includes the imageinfo plugin to help. 
    !!! imageinfo is not always accurate, use with caution, test multiple profiles !!!
    python3 vol.py -f [FILE] [windows/linux/mac].info

PROCESSES AND CONNECTIONS

  .pslist: plugin to output all current and terminated processes with their exit times,
           pulls process info from the doubly-linked list process list

  .pstree: plugin to locate processes by finding data structures that match _ESPROCESS,
           helps detect malware that unlinked itself from the process list linked list
    
  .pstree: same as pslist, but shows a tree of parent/child processes

  .netstat: attempts to list all structures in memory that have network connections
    !!! may not be stable for older versions of Windows. if it does not work, look into !!!
    !!! https://tools.kali.org/forensics/bulk-extractor for extracting PCAPs            !!!

  .dlllist: lists all DLLs associated with detected processes

______________________
HUNTING AND DETECTIONS

  .malfind: attempts to identify processes that had code injected - works by scanning heap and
            identifying processes with the executable bit (RWX/RX) set that do not have a 
            corresponding memory-mapped file on disk

  .yarascan: compares memory file against YARA rules

ADVANCED FORENSICS

  Common hooking techniques:
    SSDT Hooks
    IRP Hooks
    IAT Hooks
    EAT Hooks
    Inline Hooks

  .ssdt: System Service Descriptor Table - most common hook target, can be hooked by legit
         services so analyst discretion in what is malicious is required. 
         Windows kernel uses SSDT to look up pointers to system functions (which malware 
         can use to point to itself)

  .modules: dump list of loaded kernel modules - will miss idle or hidden files, best used 
            once potential indicators are determined as targets to look for

  .driverscan: identifies driver files in the kernel that the modules plugin may have missed,
               modules plugin should usually be used first, this often will have no output

  other useful plugins (not all of them listed are packaged with Volatility3):
    modscan
    driverirp
    callbacks
    idt
    apihooks
    moddump
    handles
