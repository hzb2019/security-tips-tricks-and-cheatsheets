Basic workflow:
  Create/open the case for the data source you will investigate
  Select the data source you wish to analyse
  Configure the ingest modules to extract specific artefacts from the data source
  Review the artefacts extracted by the ingest modules
  Create the report

Supported Disk Image Formats:
  Raw Single (For example: *.img, *.dd, *.raw, *.bin)
  Raw Split (For example: *.001, *.002, *.aa, *.ab, etc)
  EnCase (For example: *.e01, *.e02, etc)
  Virtual Machines (For example: *.vmdk, *.vhd)

Ingest Modules Docs: http://sleuthkit.org/autopsy/docs/user-docs/4.12.0/ingest_page.html

Tree Viewer Docs: http://sleuthkit.org/autopsy/docs/user-docs/4.12.0/tree_viewer_page.html
The Tree Viewer has five top-level nodes:
  Data Sources - data organised as you would typically see it in a normal Windows File Explorer. 
  Views - files will be organised based on file types, MIME types, file size, etc. 
  Results - where the results from Ingest Modules will appear. 
  Tags - will display files and/or results that have been tagged: http://sleuthkit.org/autopsy/docs/user-docs/4.12.0/tagging_page.html
  Reports - will display reports either generated by modules or the analyst: http://sleuthkit.org/autopsy/docs/user-docs/4.12.0/reporting_page.html

In Views, files may be miscategorized if they are given a misleading extension, but will be 
displayed properly when sorted by MIME type. More info: http://sleuthkit.org/autopsy/docs/user-docs/4.12.0/result_viewer_page.html

Contents Viewer: http://sleuthkit.org/autopsy/docs/user-docs/4.12.0/content_viewer_page.html
  S(core): red ! on things tagged as notable, yellow triangle on things tagged as suspicious.
  C(omment): shows a yellow page if a comment exists.
  O(ccurrence): number of times this item has appeared in previous cases (requires central repo)

Keyword searches: http://sleuthkit.org/autopsy/docs/user-docs/4.12.0/ad_hoc_keyword_search_page.html

UI docs: http://sleuthkit.org/autopsy/docs/user-docs/4.12.0/uilayout_page.html

Keyword search is in top right corner of the window.

Valuable drive information can be obtained by right clicking a data source and selecting
"View Summary Information"

On systems with limited resources it may be helpful to use the "Generate Report" function
to compile important information, then use the report to complete the investigation or 
otherwise pinpoint what exactly needs to be investigated further.

Timeline:
  The Timeline tool is composed of three areas:
    Filters: Narrow the events displayed based on the filter criteria
    Events: The events are displayed here based on the View Mode
    Files/Contents: Additional information on the event(s)

  There are three view modes:
    Counts: The number of events is displayed in a bar chart view
    Details: Information on events is displayed, but they are clustered and collapsed
    List: The events are displayed in a table view
