  "Velociraptor is a unique, advanced open-source endpoint monitoring, digital forensic and 
  cyber response platform. It was developed by Digital Forensic and Incident Response (DFIR) 
  professionals who needed a powerful and efficient way to hunt for specific artifacts and 
  monitor activities across fleets of endpoints. Velociraptor provides you with the ability 
  to more effectively respond to a wide range of digital forensic and cyber incident response 
  investigations and data breaches"
    - Rapid7 velociraptor documentation

  Velociraptor runs cross-platform between Windows, Mac, and Linux, and the binary can run 
  as both a client or server. It is also compatible with cloud storage filesystems.

CLIENT MACHINES

  To list all clients, click on the search button without any query.

  With a client machine selected, six options across the top of the page are available:
    Interrogate: Performs interrogation
    VFS: Virtual File System browser
    Collected: Shell history, VFS interaction history
    Overview: Self explanatory.
      Client ID
      Agent Version
      Agent Name
      Last Seen At
      Last Seen IP
      Operating System
      Hostname
      Release
      Architecture
      Client Metadata
    VQL Drilldown: Includes performance details, Active Directory domain, active local accounts
    Shell: PowerShell, cmd, Bash, VQL

  Creating a collection: Go to client page -> Collected tab -> Plus button in top left
    Select Artifacts: Search and choose from the given list
    Configure Parameters: Select specific artifacts to configure their parameters
    Specify Resources: Set resource usage limits for intensive tasks
    Review: View the full configuration
    Launch: Run configured collection, close config window

  Checking the Virtual File System:
    https://docs.velociraptor.app/docs/clients/vfs/

    4 folders explanation:
     "file - uses operating system APIs to access files
      ntfs - uses raw NTFS parsing to access low level files
      registry - uses operating system APIs to access the Windows registry
      artifacts - previously run collections." -THM description

    The three filesystem buttons, from left to right:
      Refresh (syncs with client)
      Recursive refresh (syncs with client)
      Recursive download

VELOCIRAPTOR QUERY LANGUAGE

  https://docs.velociraptor.app/docs/vql/

  velociraptor.exe -v query "SELECT * FROM info()"

  Artifacts (VQL Modules): https://docs.velociraptor.app/docs/vql/artifacts/
    "Velociraptor allows packaging VQL queries inside mini-programs called Artifacts. An 
    artifact is simply a structured YAML file containing a query, with a name attached to it. 
    This allows Velociraptor users to search for the query by name or description and simply 
    run the query on the endpoint without necessarily needing to understand or type the query 
    into the UI" -Velociraptor docs

  Use the ? for suggested autocompletions for a keyword

  https://docs.velociraptor.app/docs/vql/extending_vql/
